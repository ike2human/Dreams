About This Page

This page demonstrates the proper use of the HTML5 <main> element. The main element represents the dominant content of the document body.

Key Points:
  • The main element should contain the primary content of the page
  • There should only be one main element per page
  • It should not be nested inside other semantic elements like article, aside, footer, header, or nav

Content Section

This is the main content area where you would typically place your primary page content. This could include articles, blog posts, product information, or any other central content.

The main element helps screen readers and other assistive technologies identify the primary content area, improving accessibility for users with disabilities.

Additional Information

You can include multiple sections within the main element to organize your content logically. Each section can have its own heading and content.

Semantic HTML Benefits:

Using semantic elements like main, section, header, and footer makes your HTML more meaningful and accessible to both users and search engines.
