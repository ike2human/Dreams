Understanding the HTML5 Main Element

What is the Main Element?

The <main> element represents the dominant content of the document body. It contains the central topic of a document, or the central functionality of an application.

Important: There should be only one <main> element per page, and it should not be nested inside other semantic elements like <article>, <aside>, <footer>, <header>, or <nav>.

Best Practices

  • Use only one <main> element per page
  • Place it as a direct child of the <body> element
  • Don't nest it inside other semantic elements
  • Include all primary page content within the main element
  • Exclude navigation, headers, footers, and sidebars

Content Organization

Within the main element, you can organize your content using various HTML5 semantic elements:

Sections and Articles

Use <section> elements to group related content together. Each section can contain multiple paragraphs, headings, and other elements that relate to a specific topic.

Headings Hierarchy

Maintain a logical heading structure (h1, h2, h3, etc.) to create a clear content hierarchy that's easy to navigate and understand.
